# 主演示场景

## 1. 演示目标

在 12~15 分钟内展示一条完整、可验证且有技术深度的业务链路。演示不是逐页介绍，而是围绕一个客户采购需求证明：业务规则、并发正确性、事件可靠性、权限边界和可观测性确实工作。

## 2. 合成数据

演示租户：`CellarBridge Demo Trading`

客户：`锦城餐饮集团（合成）`

商品：

1. `CB-BDX-2019-750X6` — Bordeaux Blend 2019，750ml × 6；
2. `CB-BGN-2020-750X6` — Burgundy Pinot Noir 2020，750ml × 6；
3. `CB-ALS-NV-750X6` — Alsace Sparkling NV，750ml × 6。

可用路径：

- 上海一般贸易交付（抽象配置）；
- 宁波保税 B2B 交付（抽象配置）；
- 香港自由贸易交付（抽象配置）。

演示金额不使用真实市场价格，不映射真实品牌。

## 3. 演示前状态

- 客户已提交但付款条款申请为 30 天，等待经理批准；
- 三个 SKU 分别存在国内、保税和香港库存批次；
- 其中一个 SKU 的上海现货不足以满足两个并发订单；
- 路径策略 `TRP-2026-01` 已启用；
- 报价审批策略 `QAP-2026-01`：折扣 > 8% 或毛利 < 15% 或非推荐路径需要经理审批；
- 演示用户：销售、销售经理、贸易运营、仓库、财务、客户采购、审计。

## 4. 演示步骤

### 第 1 幕：客户准入和权限（约 1.5 分钟）

1. 销售登录，只能看到自己的客户和报价；
2. 打开客户申请，展示路径资格和 30 天付款条款；
3. 切换经理批准；
4. 返回销售，客户状态变为 ACTIVE；
5. 快速展示审计时间线中提交人与审批人不同。

**证明点**：OIDC、RBAC、租户/字段边界、审批审计。

### 第 2 幕：商品与供给搜索（约 1 分钟）

1. 搜索三个 SKU；
2. 展示国内现货、保税现货、香港现货和需人工确认供给的区别；
3. 指出搜索可用量不是库存承诺，页面显示更新时间。

**证明点**：领域化目录、PostgreSQL 搜索、读模型、业务语义。

### 第 3 幕：报价与可解释路径（约 3 分钟）

1. 新建报价并添加三行；
2. 输入期望日期和 30 天账期；
3. 运行路径评估；
4. 展示一条路径因 MOQ/日期被拒，其他路径分数和推荐；
5. 人工选择非推荐但有效路径，填写理由；
6. 应用 9% 折扣，系统显示审批触发原因；
7. 提交审批。

**证明点**：规则引擎不是黑盒、策略版本、金额快照、人工覆盖审计。

### 第 4 幕：审批与客户接受（约 2 分钟）

1. 经理在工作项查看触发规则和影响；
2. 批准报价；
3. 销售发送；
4. 客户门户展示不含成本/毛利的报价；
5. 客户快速双击或重复提交接受；
6. 系统只显示一个接受结果。

**证明点**：字段安全、幂等接口、状态机、并发 UI 恢复。

### 第 5 幕：订单和库存并发（约 3 分钟）

1. 系统通过可靠事件创建订单；
2. 展示同一报价只有一个订单，即使事件被重放；
3. 运行预置并发脚本，两个订单争抢同一热点批次；
4. 一个全量成功，另一个明确库存不足；
5. 数据库查询/测试报告证明 `reserved <= on_hand`；
6. 失败订单进入异常中心。

**证明点**：唯一约束、Transactional Outbox/可靠发布、原子 SQL、Testcontainers 并发测试。

### 第 6 幕：履约、恢复和结算（约 2.5 分钟）

1. 成功订单自动生成路径特定履约计划；
2. 贸易运营完成前置步骤；
3. 模拟物流节点失败，自动创建异常；
4. 执行受控重试，步骤恢复并产生公开里程碑；
5. 财务登记部分付款和剩余付款；
6. 客户门户看到公开时间线，内部审计看到完整时间线。

**证明点**：长流程编排、事件幂等、异常恢复、付款冲正模型、可见性边界。

### 第 7 幕：工程证据（约 2 分钟）

展示：

- 架构依赖测试结果；
- OpenAPI/AsyncAPI；
- 并发测试；
- trace 中的 correlation/causation；
- Grafana 业务指标；
- Git 提交和 ADR；
- README 状态与实际 Release 一致。

## 5. 备用降级演示

若完整基础设施不可用：

- 使用 core profile（PostgreSQL + Keycloak + 应用 + React）；
- Kafka 事件由持久化本地发布机制处理；
- 可观测性使用 Actuator 和结构化日志；
- 预先保留 Playwright trace、截图和测试报告；
- 不伪装组件已在线，明确说明降级模式。

## 6. 演示禁止事项

- 不输入或展示任何真实客户/公司内部数据；
- 不声称路径配置等同真实法规；
- 不现场修改数据库修复状态；
- 不跳过失败分支；
- 不展示未完成页面或伪造测试结果；
- 不把中间件数量当作主要亮点。
