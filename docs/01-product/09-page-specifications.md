# React 页面规格

## 1. 信息架构

```text
工作台
├── 经营与履约驾驶舱
├── 我的工作项
商业客户
├── 客户列表
├── 客户详情/准入审核
商品与供给
├── 酒款与 SKU 搜索
├── SKU 详情/库存供给
交易
├── 报价列表
├── 报价编辑器
├── 报价详情/客户预览
├── 订单列表
├── 订单详情
运营
├── 履约看板
├── 异常中心
财务
├── 应收列表
├── 应收详情
治理
├── 审计检索
├── 策略与模板（只读）
└── 用户与权限（简化）
客户门户
├── 报价查看与接受
└── 订单公开时间线
```

## 2. 全局交互规范

- 所有详情页显示不可变业务编号、状态、版本和最后更新时间；
- 写操作携带 `If-Match` 或版本字段，409/412 时弹出并发冲突对话框；
- 列表筛选状态写入 URL，便于分享和返回；
- 服务端状态由 TanStack Query 管理；不复制进全局 store；
- API 错误展示 `title`、稳定 `code`、`traceId` 和可操作建议；
- 所有表格支持空状态、加载骨架、错误重试和无权限状态；
- 金额显示币种；时间显示用户时区并可查看 UTC；
- 内部成本/毛利列必须由后端字段权限和前端权限共同控制；
- 删除类动作尽量替换为停用、取消、冲正；
- 演示主链路每一步提供清晰“下一动作”。

## 3. PAGE-DASH-001 经营与履约驾驶舱

**路由**：`/dashboard`

**角色**：销售经理、贸易运营、财务、租户管理员；不同角色看到不同卡片。

**组件**：

- 日期范围与租户固定上下文；
- KPI：报价数、接受率、审批中位时长、预占成功率、开放异常、逾期步骤、未收余额；
- 路径选择分布；
- 报价漏斗；
- 履约 SLA 趋势；
- 异常类型与负责人；
- 数据刷新时间和投影延迟。

**行为**：点击指标进入带筛选条件的对应列表；投影落后时显示“数据更新中”，不伪装实时一致。

## 4. PAGE-WORK-001 我的工作项

**路由**：`/work-items`

**筛选**：类型、状态、优先级、到期区间、业务编号。

**列**：标题、类型、关联对象、创建时间、到期时间、负责人、状态、快捷动作。

**特殊状态**：已被他人领取时显示最新负责人；重复领取返回冲突并刷新。

## 5. PAGE-PAR-001 客户列表

**路由**：`/partners`

**筛选**：关键字、状态、客户类型、允许路径、销售负责人。

**列**：客户编号、名称、状态、等级、默认条款、路径资格、负责人、更新时间。

**动作**：新建客户、查看、提交审核；批量删除不提供。

## 6. PAGE-PAR-002 客户详情与审核

**路由**：`/partners/:partnerId`

**页签**：概览、联系人与地址、交易资格、信用与付款条款、变更/审核历史。

**审核侧栏**：差异摘要、重复检查、提交人、审批动作。批准时确认生效版本；退回/拒绝必须填写原因。

## 7. PAGE-CAT-001 商品与供给搜索

**路由**：`/catalog`

**检索**：全文关键字、生产者、国家/产区、分类、年份、容量、供给类型、可自动预占、预计到货范围。

**结果卡/表格**：酒款、SKU、年份、箱规、供给位置、显示可用量、价格参考、供给类型、更新时间。

**边界**：不显示精确批次成本；“加入报价”打开选择客户/报价草稿对话框。

## 8. PAGE-QUO-001 报价列表

**路由**：`/quotations`

**列**：报价编号、客户、版本、状态、总额、路径、负责人、有效期、更新时间。

**快捷视图**：我的草稿、待我审批、即将过期、已接受未转订单、异常状态。

## 9. PAGE-QUO-002 报价编辑器

**路由**：`/quotations/new`、`/quotations/:id/edit`

**布局**：

1. 客户与商业条件；
2. 商品行表格；
3. 路径评估面板；
4. 价格与审批摘要；
5. 保存、评估、提交动作栏。

**商品行字段**：SKU、快照描述、数量、单位、供给偏好、列表价、折扣、净价、费用分摊、小计、验证状态。

**路径面板**：

- 推荐路径置顶；
- 每个路径显示是否有效；
- 有效路径显示成本/时效/供给/复杂度子分；
- 无效路径显示拒绝码和自然语言；
- 显示策略版本、评估时间；
- 覆盖推荐时强制填写原因。

**并发**：保存使用版本字段；冲突时提供“查看最新版本”和“复制我的改动摘要”，不自动合并金额。

## 10. PAGE-QUO-003 报价详情与审批

**路由**：`/quotations/:id`

**区域**：状态头、客户/条款、行项目、路径决策、价格摘要、审批记录、事件时间线。

**动作按状态**：编辑草稿、提交、审批、发送、撤销、客户预览。按钮必须由后端返回的 `allowedActions` 驱动或再次调用命令校验。

## 11. PAGE-PORTAL-001 客户报价页

**路由**：`/portal/quotes/:publicToken`

**公开内容**：品牌中性、商品与金额、交付方案、预计窗口、条款、有效期、接受/拒绝。

**安全**：token 不写入日志；页面不加载内部 API；过期/撤销/已处理均显示明确终态；接受按钮防重复点击，但正确性由后端幂等保证。

## 12. PAGE-ORD-001 订单详情

**路由**：`/orders/:id`

**页签**：概览、商业快照、库存预占、履约计划、应收、统一时间线。

**突出技术证据**：显示订单来源报价、幂等键摘要、预占结果、事件追踪 ID（仅系统操作员）、策略版本。

## 13. PAGE-FUL-001 履约看板

**路由**：`/fulfillment`

**视图**：按状态/到期排序的列表为默认；可选 Kanban 仅用于可视化，不改变领域模型。

**卡片**：订单、路径、当前步骤、负责人、计划时间、SLA、阻塞原因、下一个动作。

**步骤详情抽屉**：依赖、输入、公开性、历史、开始/完成/失败动作。

## 14. PAGE-EXC-001 异常中心

**路由**：`/exceptions`

**筛选**：类型、严重度、状态、负责人、模块、SLA、业务编号。

**详情**：异常摘要、检测事实、关联对象、影响、事件/日志引用、处理记录、可选恢复动作。

**恢复确认**：显示动作将影响的状态和是否可重试；高风险动作需要二次确认和原因。

## 15. PAGE-SET-001 应收详情

**路由**：`/receivables/:id`

**内容**：应收金额、已付、余额、到期日、状态、付款记录、冲正链、订单引用。

**登记付款**：金额、日期、方式（演示枚举）、外部参考号、备注；服务端返回余额；超额付款阻止。

## 16. PAGE-AUD-001 审计检索

**路由**：`/audit`

**筛选**：业务类型/编号、actor、动作、时间、correlation ID、错误码。

**结果**：时间、模块、动作、状态前后摘要、actor、来源、trace/correlation；敏感 payload 不直接展示。

## 17. 响应式和可访问性

- 主要面向 1280px 以上桌面运营场景；客户门户支持移动端；
- 表格在窄屏转为卡片或水平滚动，不能截断金额/状态；
- 焦点顺序和键盘操作经过 Playwright/axe 检查；
- ECharts 提供文本摘要和可访问标签；
- 状态不只用颜色表达。
