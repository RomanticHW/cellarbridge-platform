# 供应链与贸易交付模式分析

## 1. 建模原则

公开资料确认存在多种交付方式，但没有提供足够信息还原法律、税务和单证规则。因此 CellarBridge 使用三层抽象：

1. **Trade Route（贸易路径）：** 对客户呈现的交付方案；
2. **Supply Pool（供给池）：** 某路径可使用的库存或未来供给；
3. **Fulfillment Template（履约模板）：** 路径被选中后需要执行的步骤和 SLA。

法规、税率和实际海关代码不写死在领域枚举中，只作为可版本化政策和外部系统结果存在。

## 2. 三类演示路径

### 2.1 `SH_GENERAL_TRADE`

**业务含义：** 从上海相关一般贸易供给池向境内商业客户交付。

**演示硬约束：**

- 客户状态为 `ACTIVE`；
- 目的地在国内配送范围；
- 所有订单行在该路径允许的供给池中可获得；
- 预计交付时间不晚于客户可接受日期；
- 订单金额和数量满足路径配置。

**履约步骤模板：**

1. 库存预占；
2. 出库任务创建；
3. 拣货与复核；
4. 仓库出库；
5. 国内运输；
6. 签收。

演示不声称这些步骤覆盖真实一般贸易前置流程；MVP 假设货物已处于可交付库存状态。

### 2.2 `NB_BONDED_B2B`

**业务含义：** 从宁波保税相关供给池按商业客户资格交付。

**演示硬约束：**

- 客户拥有该路径资格；
- 客户资料和收货信息完整；
- SKU 与库存批次允许该路径；
- 订单满足配置的数量和金额范围；
- 期望日期允许额外操作时间。

**履约步骤模板：**

1. 库存预占；
2. 资料完整性检查；
3. 交付申报模拟；
4. 仓库放行；
5. 国内运输；
6. 签收。

“申报模拟”只产生合成状态，不连接政府系统，也不生成可用于真实申报的文件。

### 2.3 `HK_FREE_TRADE`

**业务含义：** 从香港供给池完成香港区域交付或由客户接收的自由贸易交付。

**演示硬约束：**

- 客户拥有香港路径资格；
- 交付区域为路径支持区域；
- SKU 可从香港供给池获得；
- 结算币种和付款条款受支持；
- 期望交付时间满足路径 SLA。

**履约步骤模板：**

1. 库存预占；
2. 放货条件确认；
3. 香港仓出库；
4. 本地运输或客户提货；
5. 签收。

## 3. 供给池模型

| 字段 | 说明 |
|---|---|
| `supplyPoolId` | 稳定标识 |
| `routeCode` | 允许使用的贸易路径 |
| `warehouseId` | 库存所在地 |
| `supplyType` | 国内现货、保税现货、香港现货、在途、海外代采 |
| `currency` | 成本计价币种 |
| `availableFrom` | 最早可承诺日期 |
| `confidence` | 对未来供给承诺的可信等级 |
| `policyVersion` | 资格和分配政策版本 |

库存事实仍由 Inventory 模块拥有；Trade Planning 只读取可承诺投影，不直接修改库存。

## 4. 路径评估

### 4.1 硬约束阶段

任何一条硬约束失败，候选路径被拒绝并返回稳定原因码，例如：

- `PARTNER_NOT_ELIGIBLE`
- `DESTINATION_NOT_SUPPORTED`
- `SKU_NOT_SUPPORTED`
- `INSUFFICIENT_PROJECTED_SUPPLY`
- `DELIVERY_DATE_UNACHIEVABLE`
- `PAYMENT_TERM_NOT_SUPPORTED`
- `MINIMUM_ORDER_NOT_MET`

### 4.2 评分阶段

只有通过硬约束的路径进入评分。默认演示权重：

| 维度 | 权重 | 说明 |
|---|---:|---|
| 预计到岸/交付成本 | 40 | 值越低得分越高 |
| 预计交付时间 | 30 | 越接近期望且风险越低得分越高 |
| 供给可信度 | 20 | 现货高于在途和待采购 |
| 运营复杂度 | 10 | 步骤越少、外部依赖越少得分越高 |

总分只用于推荐，不取代硬约束。所有输入、归一化方式、权重和结果都保存 `policyVersion`。

### 4.3 人工覆盖

销售经理可以选择非最高分的合格路径，但必须：

- 输入原因码和说明；
- 保留推荐路径、选定路径和差异；
- 生成审计事件；
- 不允许覆盖硬约束失败。

## 5. 价格构成

报价中的 `PriceBreakdown` 可包含：

- 商品基础价格；
- 国际物流估算；
- 仓储与操作费；
- 单证/服务费；
- 国内配送费；
- 税费估算占位项；
- 折扣；
- 最终净额。

各项均为演示配置，不用于真实税务或贸易决策。报价接受后保存完整快照、币种、汇率引用和计算版本。

## 6. 履约节点与外部端口

核心系统只拥有“业务承诺”和“处理状态”，不假装拥有外部机构：

| 外部能力 | 端口 | MVP 适配器 |
|---|---|---|
| 仓库出入库 | `WarehouseGateway` | 可控模拟器 |
| 承运商跟踪 | `CarrierGateway` | 固定里程碑模拟器 |
| 单证/合规 | `ComplianceGateway` | 返回合成检查结果 |
| 汇率 | `ExchangeRateProvider` | 固定版本化汇率 |
| 通知 | `NotificationGateway` | 控制台/邮件捕获器 |
| 支付 | `PaymentGateway` | 不接入；财务手工登记 |

适配器故障必须能在异常中心中观察和重试。
