## 2. 证据使用规则

- `CONFIRMED` 内容可以用于说明调研背景，但必须保留来源与“公司自述”属性；
- `INFERRED` 内容可以用于产品假设，不得写成研究对象的内部事实；
- `DEMO_ENHANCEMENT` 只用于 CellarBridge 设计，并在演示时解释其工程目的；
- 任何新增公司事实必须增加来源记录，不能凭记忆修改本表；
- 若公开资料与当前设计冲突，先更新研究和 ADR，再改变代码。

## 3. 置信度含义

| 置信度 | 含义 |
|---|---|
| 高 | 来源直接、表达明确，或由多个确认事实共同支持 |
| 中高 | 来源直接但属于企业自述，或推断链较短 |
| 中 | 符合行业规律但缺乏该公司的直接证据 |
| 低 | 不足以进入产品基线，仅可列为待验证问题 |
